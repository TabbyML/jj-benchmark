## Abandoning Commits in Jujutsu

Run the following `jj` commands from `/home/user/myproject`:

```bash
cd /home/user/myproject

# 1. Abandon `experiment`, keeping the bookmark (moves it to the parent commit A)
jj abandon experiment --retain-bookmarks

# 2. Abandon `draft`, letting the bookmark be deleted (default behavior)
jj abandon draft

# 3. Set the working-copy commit's description
jj describe -m "cleanup complete"
```

### What happens
- **Step 1:** Commit `B` is discarded; descendants (`C`, `D`) are rebased onto `A`. Because of `--retain-bookmarks`, the `experiment` bookmark is moved to `A` (alongside `feature-a`) instead of being deleted.
- **Step 2:** Commit `C` is discarded; descendant `D` is rebased onto its new parent. Without `--retain-bookmarks`, the `draft` bookmark is removed.
- **Step 3:** The current working-copy commit (the rebased `D`) has its description replaced with `cleanup complete`.

Verify the resulting state with:

```bash
jj log
jj bookmark list
```

You should see `feature-a` and `experiment` on commit `A`, no `draft` bookmark, and the working copy carrying the `cleanup complete` description with `d.txt` still present.
