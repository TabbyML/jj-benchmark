# Working Copy as Commit ## Background In Jujutsu (`jj`), the working copy is always a permanent commit. Any file change is automatically recorded in the current commit when you run any `jj` command. This eliminates the need for a separate staging area and makes it easy to track changes. ## Requirements 1. Initialize a new Git-compatible jj repository in the `/home/user/repo` directory. 2. Configure your identity for the repository (e.g., `jj config set --user user.name "Test User"` and `jj config set --user user.email "test@example.com"`). 3. Create a file named `hello.txt` containing the text `Hello jj`. 4. Create a new empty working copy on top of your current changes (leaving the changes to `hello.txt` in the previous commit). ## Constraints - The repository must be located at `/home/user/repo`. - The file `hello.txt` must be committed in the parent of the current working copy (`@-`). - The current working copy (`@`) must be empty.
