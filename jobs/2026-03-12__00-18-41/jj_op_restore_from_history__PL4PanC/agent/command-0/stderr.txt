| | Git | jj |

| Recover | `git reset --hard <sha>` | `jj op restore <op-id>` |
| Destructive? | Yes — rewrites HEAD | No — adds a new operation on top |

`jj op restore` is strictly non-destructive: it never removes entries from the operation log. The squash operation `0a3c4d92b06a` still exists and could itself be restored to if needed.
