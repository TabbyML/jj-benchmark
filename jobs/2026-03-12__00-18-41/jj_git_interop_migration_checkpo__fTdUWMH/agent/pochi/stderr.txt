### Key concept: co-located repos

In a **co-located** repository (`jj git init --colocate`), jj and Git share the same object store. Every time a jj bookmark advances, it is immediately reflected as a Git branch — **no manual `jj git export` is needed** for day-to-day work. Your Git-using teammates can simply `git fetch` / `git checkout jj-migration` as they normally would.
